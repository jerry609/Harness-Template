# Golden Principles

Use these principles to keep the repository agent-friendly.

1. Repo-local knowledge beats chat history.
2. A thin router beats a giant instruction dump.
3. Specs define intent; plans define execution.
4. One session should advance one coherent slice of work.
5. Outcome beats narration.
6. Human review feedback should become automation when repeated.
7. Tools should be narrow, explicit, and easy to call correctly.
8. Cleanup is part of delivery, not a separate phase.
9. Small diffs are easier to verify, review, and resume.
10. The repository should remain legible to the next agent run.
